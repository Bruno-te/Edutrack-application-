# 📚 Student Performance Monitor (SPM)
### Flutter + BLoC + Firebase Firestore

A full-featured, production-ready mobile app to track student grades, attendance, assignments, and performance analytics — with role-based access for **Admins**, **Teachers**, **Students**, and **Parents**.

## 🔥 Firebase Collections

| Collection      | Description                              |
|-----------------|------------------------------------------|
| `users`         | All user profiles (role, name, email)    |
| `grades`        | Grade records per student/subject/term   |
| `attendance`    | Daily attendance per student/subject     |
| `assignments`   | Assignments created by teachers          |
| `submissions`   | Student submissions per assignment       |
| `notifications` | In-app notifications per user            |

## 🚀 Setup & Installation

### Prerequisites
- Flutter SDK ≥ 3.3.0
- Dart SDK ≥ 3.3.0
- Firebase project with Authentication + Firestore enabled

### Step 1 — Clone and install
```bash
git clone <your-repo>
cd spm_app
flutter pub get
```

### Step 2 — Connect Firebase
```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Configure (generates firebase_options.dart automatically)
flutterfire configure
```

### Step 3 — Deploy Firestore Rules & Indexes
```bash
# Install Firebase CLI if needed
npm install -g firebase-tools
firebase login

# Deploy rules
firebase deploy --only firestore:rules

# Deploy indexes
firebase deploy --only firestore:indexes
```

### Step 4 — Enable Auth methods
In Firebase Console → Authentication → Sign-in methods → Enable **Email/Password**

### Step 5 — Run the app
```bash
flutter run
```

## 🌱 Seeding Test Data

Edit `lib/core/utils/seed_data.dart` and replace:
```dart
static const String _sampleStudentId   = 'STUDENT_UID_HERE';  // From Firebase Auth
static const String _sampleTeacherId   = 'TEACHER_UID_HERE';
```

Then call once in `main()`:
```dart
await SeedData.seed();
```

Remove/comment after seeding to avoid duplicates.
